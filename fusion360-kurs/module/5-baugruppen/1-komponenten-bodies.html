Komponenten vs. Bodies

⏱ 35 Minuten 🎯 Baugruppen 🌳 Strukturierung

Lernziele

  • Ich verstehe den Unterschied zwischen Body und Component.
  • Ich entscheide bewusst, ob etwas als Body oder als Component modelliert wird.
  • Ich konvertiere Bodies nachträglich in Components.
  • Ich nutze Hierarchien – Components in Components – für komplexe Baugruppen.
  • Ich kenne den Unterschied zwischen internen und externen Components.

Vorwissen-Check

Was war ein Body bisher in unserem Kurs?
Ein 3D-Klumpen Material – Ergebnis von Extrudieren, Revolve, Loft etc. Bodies haben keine eigene Origin, keine eigenen Eigenschaften, leben in der Origin der Datei (oder Component).
Was haben wir mit Components in Modul 3 und 4 gemacht?
Mehrere Bauteile (Wandplatte + Aufnahme; Boden + Seiten + Rückwand) als getrennte Einheiten organisiert, jede mit eigener Origin und Eigenschaften, damit BOMs sinnvoll werden.

Wann reicht ein Body?

  • Einzelteile, die nie woanders gebraucht werden (z. B. ein Spielzeugauto-Vollkörper).
  • Hilfskonstruktionen, die du gleich wieder löschst.
  • Frühe Skizzenphase – „erst Volumen, dann Struktur".

Wann brauchst du Components?

  • Mehrere Bauteile, die zusammengebaut werden.
  • Stücklisten und Drawings.
  • Bewegungen (Joints, Modul 5.2).
  • Wiederverwendbare Bauteile in mehreren Projekten (Schrauben, Beschläge, …).

Bodies in Components umwandeln

  1. Browser-Rechtsklick auf den Body → Create Components from Bodies.
  2. Fusion erzeugt eine neue Component, packt den Body hinein, verschiebt die Origin auf die Body-Origin.
  3. Anschließend Component umbenennen, Material zuweisen.

Hierarchien

Components können andere Components enthalten – beliebig tief. Sinnvolle Strukturierung: 

Wandregal (Top-Component)
├── Korpus
│   ├── Boden
│   ├── Seite-links
│   ├── Seite-rechts
│   └── Rueckwand
├── Befestigung
│   ├── Cleat-Wandseite
│   └── Cleat-Regalseite
└── Inhalt (optional)
    ├── Box-1
    └── Box-2

So kannst du z. B. Korpus komplett ein-/ausblenden, oder eine BOM nur über Befestigung erstellen.

Interne vs. externe Components

  • Intern: Component lebt in der aktuellen Datei. Standard.
  • Extern (Distributed Design oder Component Reference): die Component verweist auf eine separate Datei. Mehrere Designs nutzen dieselbe Component – Änderung an der externen Datei propagiert sich überall.

Externe Components sind perfekt für Standardteile (M3-Schraube, Heat-set-Insert, Spax-Schraube). Einmal gut konstruiert, in jedem Projekt referenziert. Erstellung: Component im Browser rechtsklicken → Save as Component → eigene Datei.

Anti-Pattern „Alles als Bodies" – sobald die Datei mehr als 3 Bauteile hat oder du an Drawing/BOM denkst, kostet die Umstellung später Zeit. Lieber von Anfang an in Components arbeiten.

Geführte Übung – Wandhalterung umstrukturieren

Wir nehmen das Modul-3-Projekt und strukturieren es zu einer sauberen Component-Hierarchie.

  1. Wandhalterung öffnen. Lektion-3-6-Wandhalter.
  2. Top-Component anlegen. Falls noch nicht: Browser-Rechtsklick auf den Top-Eintrag → New Component → Wandhalter. Beide Bauteile (Wandplatte, Aufnahme) per Drag-and-drop im Browser unter Wandhalter einsortieren.
  3. Sub-Hierarchie. Innerhalb von Wandhalter zwei Sub-Components: Befestigung und Aufnahme-Set. Wandplatte in Befestigung verschieben. Aufnahme in Aufnahme-Set.
  4. Standardteil als externe Component. Konstruiere eine M4-Senkkopf-Schraube (DIN 7991, 25 mm lang) als eigenständige Component. Anschließend Browser-Rechtsklick → Save as Component. Speichern als Standard/M4-Senkkopf-25.f3d.
  5. Schraube referenzieren. Im Wandhalter-Design die externe Schraube referenzieren: Toolbar Insert → Insert Derive oder im Data Panel die Schraube reinziehen. Sie taucht als externe Component auf. Zwei Mal: für jede Wandbohrung eine Instanz, an die jeweilige Position joinen (Joint-Lektion folgt).
  6. Hierarchie prüfen. Browser sollte jetzt etwa so aussehen: 
    Wandhalter
├── Befestigung
│   ├── Wandplatte
│   ├── M4-Senkkopf-25 (extern) × 2
└── Aufnahme-Set
    └── Aufnahme
  7. BOM aktualisieren. Drawing öffnen, BOM rechtsklicken → Refresh. Die Schrauben sollten jetzt mit Anzahl 2 in der Liste auftauchen.
  8. Speichern.
Erwartetes Ergebnis Sauber strukturierte Hierarchie mit Sub-Components, eine externe Standardteil-Component, BOM zeigt Schrauben automatisch.

Freie Transferaufgabe – Standard-Bibliothek aufbauen

Lege im Data Panel einen neuen Ordner Standardteile an und konstruiere dort drei externe Components, die du in jedem zukünftigen Projekt verwenden kannst:

  • M3-Linsenkopf-12 (DIN 7985).
  • M4-Senkkopf-25 (DIN 7991).
  • Heat-set-Insert-M3 (Standardmaße aus Lektion 3.3).

Akzeptanzkriterium: Drei eigenständige Fusion-Files. In einem beliebigen Test-Design alle drei via Insert verfügbar.

Selbstkontroll-Quiz

1. Was hat eine Component, was ein Body nicht hat?
Eigene Origin, Materialzuweisung, Properties (Part Number, Description), Joint-Fähigkeit, BOM-Eintrag, Möglichkeit als externe Datei zu existieren.
2. Wie verwandelst du einen Body in eine Component?
Browser-Rechtsklick → Create Components from Bodies. Fusion legt eine neue Component an, der Body landet darin, Origin wird gesetzt.
3. Wann lohnen sich externe Components?
Bei Standardteilen (Schrauben, Inserts, Beschläge), die in vielen Projekten genutzt werden. Eine Änderung am Original aktualisiert alle Verwendungen.
4. Welche Hierarchietiefe ist sinnvoll?
2–3 Ebenen sind meist klar lesbar. Mehr Tiefe hilft nur bei sehr komplexen Baugruppen (z. B. Maschinenbau). Ziel: ein Außenstehender versteht die Struktur in 30 Sekunden.
5. Was passiert mit der Origin, wenn du einen Body in eine Component umwandelst?
Fusion legt eine eigene Component-Origin an, deren Lage der Body-Position folgt. Diese Origin ist später nützlich für Joints und Spiegelungen.

Häufige Stolperer

  • Operationen werden in falscher Component erstellt → Active-Component prüfen (kleiner Punkt vor dem Namen). Beim Skizzieren immer erst aktivieren.
  • BOM zeigt nichts an → Top-Level besteht nur aus Bodies, nicht Components. Bodies in Components umwandeln.
  • Externe Component lässt sich nicht bearbeiten → vorher Open auf der externen Datei, dort ändern, speichern, Master-Datei aktualisiert sich.
