Export: STL, 3MF, Mesh-Workspace, Reparatur

⏱ 35 Minuten 🎯 3D-Druck-Fokus 📤 Slicer-Übergabe

Lernziele

  • Ich kenne den Unterschied zwischen STL, 3MF und OBJ.
  • Ich wähle die richtige Refinement-Stufe für meinen Anwendungsfall.
  • Ich exportiere einzelne Bodies oder ganze Components getrennt oder gemeinsam.
  • Ich kenne den Mesh-Workspace und kann importierte STL reparieren.
  • Ich übergebe sauber an PrusaSlicer, Cura, Bambu Studio o.ä.

Vorwissen-Check

Was ist eine Mesh-Datei?
Eine 3D-Datei, die die Oberfläche eines Bauteils als Netz aus Dreiecken beschreibt. STL und OBJ sind klassische Mesh-Formate. CAD-Modelle (B-Rep) hingegen beschreiben Geometrie mathematisch exakt.
Warum reicht für FDM meistens „Medium"-Refinement?
Drucker mit 0,4 mm Düse können Details unter 0,2 mm sowieso nicht abbilden. Höhere Refinement-Stufen produzieren riesige Dateien, die der Slicer langsam lädt, ohne dass der Druck sichtbar besser wird.

Theorie kompakt

Formate im Überblick

STLStandard im Hobbybereich. Reine Geometrie, keine Farbe, keine Einheit (oft mm angenommen). Binary kleiner als ASCII.
3MFModernes Format, transportiert Farbe, Maßeinheit, Multi-Material, sogar Slicer-Settings. Wird zunehmend von allen Slicern unterstützt.
OBJAus der 3D-Grafik-Welt. Im 3D-Druck weniger üblich, aber unterstützt Texturen.
STEPCAD-Format, kein Mesh. Manche Slicer lesen es direkt (PrusaSlicer ja). Vorteil: keine Triangulierung nötig, exakte Geometrie.

Refinement-Stufen

  • Low: grob, sichtbare Polygonkanten. Nur für sehr einfache Bauteile.
  • Medium: Empfohlen für FDM. Glatt genug, Datei klein.
  • High: für sichtbar runde Flächen, sehr feine Details. Datei wird ca. 4–8× größer als Medium.
  • Custom: eigene Werte für Surface Deviation, Normal Deviation, Maximum Edge Length, Aspect Ratio.

Wie exportiere ich aus Fusion?

Drei Wege, alle zum gleichen Ergebnis:

  1. Browser-Rechtsklick auf einen Body oder eine Component → Save As Mesh. Schnellster Weg für Einzelteile.
  2. Datei → Export: ganze Datei als 3MF/STL.
  3. Manufacture-Workspace → 3D Print: dortiges Werkzeug zeigt Vorschau, kann direkt in einen Slicer launchen, wenn die App-Verknüpfung gesetzt ist.

Mehrere Bodies, eine Datei

Im Save-as-Mesh-Dialog kannst du One file oder One file per body wählen. Für Multi-Material- oder Multi-Color-Drucke (3MF mit Materialzuordnung) hilft One file. Für getrenntes Drucken einzelner Bauteile One file per body.

Der Mesh-Workspace

Wenn du eine STL importierst, landet sie als Mesh-Body im Browser. Im Mesh-Workspace (Toolbar oben links auf Design → Mesh) kannst du:

  • Repair: Löcher schließen, falsche Normalen korrigieren.
  • Reduce: Polygone reduzieren (für simplere Geometrie).
  • Modify → Convert Mesh: Mesh in einen Solid Body umwandeln (begrenzt sinnvoll, oft zu komplex für Fusion).

Wichtig: Mesh-Bodies haben keine Timeline-Geschichte. Bearbeiten sie ist nicht parametrisch. Im Zweifel lieber neu konstruieren statt importierte Meshes umbauen.

Slicer-Schnittstelle PrusaSlicer und Bambu Studio öffnen 3MF nativ inklusive Komponentenfarben. Cura behandelt Multi-Body in 3MF unterschiedlich. Wenn der Slicer „verschiedene Bauteile" erkennt, aber alles als ein Drucker-Job sehen soll: in Cura per Rechtsklick „Merge Models" zusammenführen.

Geführte Übung – sauberer Export

  1. SnapBox aus Lektion 3.4 öffnen.
  2. STL für Einzelteile. Browser-Rechtsklick auf den Box-Body → Save As Mesh. Dialog:
    • Format: STL (Binary).
    • Refinement: Medium.
    • Speicherort: dein Slicer-Eingangsordner.
    • Dateiname: SnapBox-Box.stl.
    OK. Wiederhole für den Deckel-Body.
  3. 3MF für gemeinsamen Druckjob. Datei → Export → Format 3MF. Speicherort, Name SnapBox-Set.3mf. OK. Diese Datei lässt sich in einen Slicer laden, beide Bauteile sind sofort getrennt platziert.
  4. Refinement-Vergleich. Exportiere die Box dreimal: einmal als Low, einmal Medium, einmal High. Vergleiche die Dateigrößen im Dateimanager. Beobachtung: bei einer einfachen Box hilft High kaum, kostet aber Speicher.
  5. Slicer-Test. Lade SnapBox-Box.stl in deinen Slicer. Schau dir die Druckvorschau an: liegen Federarme flach? Schichtorientierung wie geplant? Wenn nein: im Slicer rotieren, oder in Fusion das Design im CAD anders orientieren und neu exportieren.
  6. Mesh-Reparatur (Demo). Lade in einem neuen Fusion-Design eine fremde STL (z. B. ein heruntergeladenes Modell von Printables). Wechsle in den Mesh-Workspace. Modify → Inspect zeigt eventuelle Defekte. Modify → Repair schließt offene Kanten. Den parametrischen Workspace verlässt du dabei – Reparatur hat keine Timeline.
Erwartetes Ergebnis Du hast eine STL und eine 3MF im Slicer-Ordner, kennst die Größenunterschiede der Refinement-Stufen, und hast einmal den Mesh-Workspace betreten und wieder verlassen.

Freie Transferaufgabe – Multi-Color-Druck vorbereiten

Wenn du einen Drucker mit AMS / MMU hast: Exportiere den Aufbewahrungswürfel aus Modul 1 als 3MF. Vor dem Export im Browser den Body in zwei Components aufteilen (Boden + Wände) – dazu rechtsklicken auf den Body, Create Components from Bodies – und im Slicer beiden Components verschiedene Farben zuweisen.

Akzeptanzkriterium: 3MF-Datei Wuerfel-zweifarbig.3mf, in deinem Slicer landen zwei Komponenten mit getrennt zuweisbaren Farben.

Selbstkontroll-Quiz

1. Welche Refinement-Stufe ist für FDM meist optimal?
Medium. High macht die Datei groß, ohne dass es im Druck (0,4er Düse, 0,2 mm Schicht) sichtbar wird.
2. Wann lohnt sich 3MF gegenüber STL?
Wenn Multi-Material/Color, Maßeinheiten oder Slicer-Settings transportiert werden sollen. Auch wenn mehrere Bodies als ein „Set" erhalten bleiben sollen.
3. Was ist der Unterschied zwischen Mesh-Body und Solid-Body?
Solid-Body = exakte CAD-Geometrie, parametrisch editierbar. Mesh-Body = Dreiecksnetz, nur grob bearbeitbar, keine Timeline.
4. Warum sollte man importierte STL nicht in Mesh-Body weiterbearbeiten?
Mesh-Bearbeitung ist nicht parametrisch und produziert oft komplexe, schwer reparierbare Geometrie. Wenn das Original nicht passt, eher neu konstruieren als das Mesh umbauen.
5. Welche Slicer-Software unterstützt 3MF nativ?
PrusaSlicer, Bambu Studio, Orca Slicer, neueres Cura. STL ist der kleinste gemeinsame Nenner und immer eine sichere Wahl, wenn Slicer-Kompatibilität unklar ist.

Häufige Stolperer

  • STL ist ein Bruchteil zu groß im Slicer → Slicer hat „Inch" als Einheit angenommen. STL hat keine Einheit. Slicer-Setting prüfen.
  • Datei „explodiert" zu hunderten MB → Refinement zu hoch. Auf Medium reduzieren.
  • Slicer findet „Multi Mesh" → mehrere Bodies in einer STL. Im 3MF macht das Sinn, in STL eher nicht. Lieber One file per body exportieren.
